📋 JSON Schema Reference

You can hand-edit single-resume JSON files and import them with ⬆ Import One. Exported files wrap the resume in a container object; you can also import a bare data object using just the fields below.

Minimum viable shape (paste into a .json file and import):

{
  "fullName": "Jane Doe",
  "tagline": "Full-Stack Engineer  |  3+ Years",
  "location": "Mumbai, India",
  "phone": "+91 9999999999",
  "email": "jane@example.com",
  "linkedin": "linkedin.com/in/jane-doe",
  "portfolio": "jane.dev",
  "github": "github.com/jane-doe",
  "summary": "Full-stack engineer with experience in...",
  "skills": [
    { "category": "Languages", "items": "TypeScript, Python, Go" },
    { "category": "Frameworks", "items": "React, Next.js, FastAPI" }
  ],
  "experience": [
    {
      "title": "Senior Engineer",
      "company": "Acme Corp",
      "location": "Remote",
      "dates": "Jan 2024 – Present",
      "projects": [
        {
          "name": "Project X",
          "tagline": "Internal analytics platform",
          "bullets": [
            "Built a real-time dashboard used by 200+ users.",
            "Cut query latency by 60% with Redis caching."
          ],
          "tech": "TypeScript, React, FastAPI, Redis, PostgreSQL"
        }
      ]
    }
  ],
  "projects": [
    {
      "name": "Weather App",
      "tagline": "Cross-platform Flutter weather app",
      "bullets": [
        "Built a weather app in Flutter with real-time forecasts.",
        "Integrated public APIs via Dio with full error handling."
      ],
      "tech": "Flutter, Dart, Dio, http, REST APIs"
    }
  ],
  "education": [
    { "degree": "B.Tech Computer Science", "school": "IIT Delhi", "year": "2021" }
  ]
}

Tips:

  • All fields are optional — leave arrays empty ([]) or strings empty ("") if you don't need them.
  • bullets is an array of strings, one bullet per entry.
  • A job's projects array can hold one or many projects.
  • The top-level projects array (outside experience) renders as a separate "Projects" section — useful for portfolio work, open source, or learning projects.
  • To skip the Projects sub-structure for a job, put the achievements directly into a single project with an empty name and tagline — only its bullets will render under the job.
  • Import accepts both full export format ({"type":"resume_builder_single_resume","resume":{...}}) and bare data shown above.

☁ Cloud Sync Settings

Sync your resume history across devices using a private GitHub Gist. Your data is stored in your own Gist — GitHub never sees the app, and the app never sees your GitHub account beyond this one token.

First-time setup (≈ 3 minutes)
  1. Go to github.com/settings/tokens → Generate new token (fine-grained).
  2. Name it "Resume Builder Sync". Set expiration (up to 1 year is fine).
  3. Under Account permissions, find Gists → set to Read and write. Leave everything else as "No access".
  4. Generate and copy the token (starts with github_pat_). Paste it below.
  5. Go to gist.github.com → create a new Secret gist with filename resume_builder.json and content {}. Click Create secret gist.
  6. Copy the Gist ID from the URL (the long string after your username). Paste it below.
  7. Click Save & Test. If green, you're set.
